JavaScript面向对象的程序设计-类的创建

发表于 2017-01-04 | 阅读次数

面向对象的编程语言都有一个特征，就是拥有类的概念，通过类可以创建任意多个具有相同属性和方法的对象。然而众所周知，ECMAScript中没有类的概念，因此它的对象也与其他基于类的语言中的对象有所不同。

JavaScript中用对象模拟其他语言中的类，有下面几种模式对类进行创建。

构造函数模式

ECMAScript中的构造函数能够创建特定类型的对象，像Array，Object这样的原生构造函数，我们也可以创建自定义的构造函数。

Person构造函数就相当于Person类的构造函数一样的存在，当需要创建Person实例时，需要使用new操作符，实例生成步骤如下。

  1. 创建一个新的对象
  2. 将构造函数的作用域赋给新对象，this指向新对象
  3. 执行构造函数中的代码，为新对象添加属性方法等
  4. 返回新对象
    构造函数的问题: 上个栗子中有一个sayName的方法，每new一个Person都会产生一个新的实例，一个新的sayName方法。创建两个功能相同的方法实在是没有什么必要，为此就有了下面一种方法”原型模式”

原型模式

我们每创建一个函数都会有一个prototype(原型)属性，这属性是一个指针，指向一个对象，这个对象的作用就是包含由特定类型的所有实例共享的属性和方法。使用原型的好处是可以让所有实例共享构造函数的属性和方法，换句话说就是将构造函数模式中定义在构造函数内的属性和方法定义在构造函数的prototype上。

创建了自定义构造函数Person()后，其原型对象会自动生成一个constructor指向Person。当调用构造函数生成实例后，实例会有一个[[prototype]]属性，虽然这个属性在js中没有明确的访问方式，不过Firefox、chrome和Safari都有一个proto表示，实例的[[prototype]]指向构造函数的prototype。

然而原型模式也存在着问题，定义在prototype上的函数或方法是被所有的实例所共享的。如果是值类型的属性倒也还好，当属性为引用类型时就会出现问题。

所以就有了下面这种创建类的模式

组合使用构造函数和原型模式

组合使用构造函数和原型可以说是创建自定义类型比较常见的方式，使用构造函数来定义实例属性，原型模式定义方法和共享的属性。每个实例都有自己的实例属性，同时又共享对方法的引用，最大限度的节省了内存。同时这种方法还能够向构造函数传递参数，可谓集两者之所长。

两个实例都拥有各自的name和brother，以及共享的sayName方法。

动态原型模式

创建类的时候需要将有独立的构造函数和原型，似乎不太合理，动态原型模式使得这一问题得以解决。它把所有的信息都封装在了构造函数中，既能通过在构造函数中检查并初始化原型，又能保持组合创建的优点。

寄生构造函数模式

–也是第一次听到这个模式，书上说在前几种模式不适用的情况下可以采用这一种模式。思想是创建一个函数，函数的作用仅仅是封装创建的代码，然后返回新创建的对象。

这个模式可以在特殊情况下为对象创建构造函数。假设我们要创建一个具有额外方法的特殊数组，因为不能直接改造Array，可以使用这个模式。

关于寄生构造函数模式，返回的对象和构造函数、构造函数原型属性之间没人有何关系，也就是说函数返回的对象和其外部创建的对象没什么区别。在可以用其他模式的情况下不要使用这种模式

稳妥构造函数模式

稳妥构造函数和寄生构造函数模式类似，但有亮点不同:一是新创建的对象实例方法不引用this，二是不适用new调用构造函数。

在这种模式创建的对象中只有toPipedString方法能访问name，没有别的办法访问。这种安全性使它适用于某些安全执行环境。

Cookie、sessionStorage、localStorage的区别

Cookie早期用来存放用户的相关信息，例如用来判断用户是否登陆等等。一般由服务器进行生成，能够设置有效时间等；在与服务器通信时携带在http头中，大小在4kb左右。
sessionStorage和localStorage都是HTML5中加入的技术，两者的作用和cookie相同均用来保存数据到浏览器本地，两个Storage大小在5M左右。
localStorage将数据保存在浏览器本地，除非手动清除否则将会永久存在；
sessionStorage用来保存会话中的数据，一旦会话结束那么数据也会被清除。
两者都仅仅在浏览器中保存数据，不会参与同服务器的通信。

JSONP原理

JSONP是JSON With Padding的简称。因为浏览器支持同源策略 —— 只有满足同源的脚本才会被执行，所谓同源就是域名、协议和端口都相同。而我们有的时候会将脚本放置在不同的服务器，这时就需要进行Js的跨域。JSONP就是利用“script”标签不受同源策略的限制来进行跨域的，通过javascript callback的形式进行单项跨域跨域访问。欲知详情，请点这里。

闭包

javascript中当一个对象的引用被全部解除后，该对象就会被回收。内部作用域中的变量在方法执行完成，引用全部解除后被销毁从而导致外部作用域无法对其进行访问。通过闭包能将变量始终保持被引用的状态，实现外部变量对内部变量的永久访问；与此同时闭包也会增加内存的开销。

说说前端跨域的解决方式

由于同源策略的限制，在非同源的情况下以下三种情况不能会受到限制。

  • cookie、localstorage和indexDB无法读取
  • DOM无法访问
  • AJAX请求不能发送

    document.domain

    同源的网页能够共享cookie，但是一级域名相同，二级域名不同的网页则会收到同源策略的限时。浏览器允许通过设置document.domain来共享cookie，此外服务器设置cookie的时候指定一级域名，这样二级三级域名不用做任何设置就能访问。

    window.postMessage

    H5中引入一个新的API:跨文档通信API，这个API在window对象新增了一个postMessage方法，允许跨窗口通信，无论是否同源。能够实现对localstorage的跨域访问

    AJAX跨域

    ajax有三种方法实现跨域访问：JSONP,WebSocket,CORS。

    参考:浏览器同源策略及其规避方法

JSONP

JSONP就是通过在插入一个”script”元素，向服务器请求JSON数据，script不受同源策略的限制；服务器收到后将数据放在一个指定名字的回调函数中传回来。

WebSocket

一种通信协议，使用ws://(非加密),wss://(加密)作为协议前缀。此协议不实行同源策略，只要服务器支持就能进行跨域通信。

CORS - 跨源资源分享（Cross-Origin Resource Sharing）

它允许浏览器向跨源服务器，发出XMLHttpRequest请求，从而克服了AJAX只能同源使用的限制。

神奇的负值Margin

发表于 2016-12-27 | 阅读次数

以前对于负值Margin一直有一种迷之自信，认为相当于一个反方向的属性而已，直到最近复习一些布局的时候才发现自己是多么的孤陋寡闻。感谢大漠的负margin用法权威指南以及无双的CSS布局奇淫巧计之-强大的负边距两篇文章让我对负值margin和布局有更深入的了解

什么是负margin

Css中margin表示元素与其相邻元素之间的距离，负margin也就是margin的取值为负，一个负值margin长这样

以前很少用负margin值，因为不了解所以根本不会想到。负margin其实能做很多事情，下面是负margin几个注意的地方。

  1. 负margin是绝对标准的CSS
  2. 不脱离文档流！负margin不会破坏页面的文档流，当你使用负margin上移一个元素的时，其后的元素会紧接着上移。
  3. 完美的兼容性：几乎所有现在浏览器都支持负值margin，甚至连IE6在大多情况下也能够支持
  4. Dreamweaver不会在设计试图上解析负margin，不过我想现在应该没多少人用DW了吧

负值margin对state元素的影响

state元素也就是标准文档流下的元素，没有设置浮动、定位等属性，margin有上左下右四个方向的值可以进行设置。
如果元素没有设置width属性或者宽度为auto时，margin-left、margin-right的负值会改变元素的宽度，也就是向左向右增加。
元素指定宽度的时影响效果如下:

  1. 当margin-top、margin-left设置了负margin时，元素会像左边👈或上边👆移动margin的值个单位；
  2. 当设置margin-right、margin-bottom的时候，元素并不会像top、left一样移动元素，而是将后面元素向左向上移动margin值个单位，这样就会后面的元素上移，间距减小，从而可能覆盖当前元素。css会认为元素的宽度减少了，然后让后面的元素补位，元素实际的宽度并没有改变。

负值margin对浮动元素的影响

负margin对浮动元素和state元素相差的不多，同样的也是移动元素或者是移动后面的元素。仅仅有一点区别，根据元素浮动的方向不同，margin-left、margin-right产生的效果也会有所区别。

两个div均为左浮动，此时margin-left使得div1向左移动了10px，由于margin-right的原因，div2向左移动了20个像素;
当div为右浮动时，margin-right让div1向右移动了20像素，margin-left使div2移动了10px。

总之，负margin对浮动元素的影响根据浮动的方向而变化，当设置与浮动方向相同的负margin时，能够让元素向浮动方向移动margin的值个单位；设置与浮动方向相反的负margin属性时，减少元素在流上的宽度，其后的元素会进行补位，覆盖前一个元素。

对绝对定位元素的影响

由于绝对定位的top、left等值是指元素自身的边界到最近的以设置定位属性的祖先元素的距离，当margin为正时，元素的边界会向外扩张；为负值时向内收缩。

ES6学习笔记-Set和Map数据结构

发表于 2016-12-17 | 阅读次数

无论在任何编程语言中，数据结构都是极其重要的基础，采用正确的数据结构甚至可以直接影响开发者的开发效效率

原有的数据结构

​ 在ECMAScript中，定义了以下几种数据结构，其中分为值类型(Primitive Type)、引用类型(Reference Type)

值类型数据结构：

  • String - 字符串
  • Number - 数值
  • Boolean - 布尔
  • Null - 空值
  • Undefined - 未定义值

引用类型数据结构：

  • Object - 对象
  • Array - 数组
  • RegExp - 正则表达式
  • Data - 日期
  • Error - 错误

在严格意义上来说、引用类型只有Object一种。因为Array、Data等类型都是派生于Object的子类

ES6中新增的数据结构

Set集合

​ 在ECMAScript中，Array表示一系列元素的有序集合，每一个元素都带有一个自然数作为下标表示此元素的位置。但在很多情况下我们可能并不关心集合的顺序，甚至需要一个无序集合，在ES6之前这一直是一块空缺，直到TC-39将无序集合(Set)加入到ECMAScript中从而得以完善。

使用语法

Set无法像数组一样通过”[]”语法直接生成，需要使用新建对象的方法来创建;在创建的时候我们也可以直接使用一个数组对象作为该集合对象的初始元素。

操作方法

由于现在没有直接的语法糖支持集合、所以对集合的操作都要通过集合对象来实现

  • set.add(value) —— 添加元素到集合内
  • set.delete(value) —— 删除集合内的指定元素
  • set.has(value) —— 判断集合内有无指定元素
  • set.clear() —— 清空集合内的所有元素
  • set.forEach(callbackFn[, context]) —— 遍历集合内的元素、传入一个回调函数

数组与集合的区别

数组(Array) 集合(Set)
元素列表 有序 无序
元素可重复性 可重复 不可重复

因为集合是无序的，所以不能像数组一样通过下标获取元素；但由于集合是可迭代的对象，所以可以使用ES6中的for of 对集合进行迭代，当然使用forEach也是可行的。

集合的不可重复性

Map映射类型

映射类型有很多种称呼，例如Map、Hash等，ECMAScript中的Object在结构上也属于映射类型。映射类型在计算机科学中的定义为关联数组，关联数组的定义是由若干个键值对组成的集合，每个键(Key)都是唯一的

使用语法

和Set一样Map也不能通过”{}”语法来直接创建，需要创建一个实例；创建实例的时候可以通过传入一个二维数组，Map根据数组的顺序对实例进行初始化，如果有重复的键则会按照FIFO(First In First Out)规则，以最后一个处理的Key最终值。

操作方法

  • map.set(key, value) //添加键值对到map中
    • map.get(key) //获取指定key的value
  • map.delete(key) //从map中移除指定key的键值对
  • map.clear() //清空map中的所有键值对
  • map.entries() //将map的键值对以二维数组的形式返回
  • map.has(key) //判断map中是否存在指定key的键值对
  • map.keys() //返回map中的所有键(key)作为一个可迭代对象
  • map.values() //返回map中的所有值(value)作为一个可迭代对象
  • map.size //map映射中所有键值对的数量

    遍历映射对象中的键值对

    由于映射对象是关联数组的一种实现，所以映射对象也同样是一种可迭代的对象。可以通过for-of来对其键值对进行遍历，当然也可以用ForEach来进行遍历

Map映射类型和Object的区别

Map和Object在很多方面都是相同的，Map比Object多了两个非常方便的地方

  1. 可以使用任何对象作为键
  2. 通过size属性可以方便的知道键值对的数量

    除了这点之外、映射对象和普通对象的区别还体现在JSON序列化结果中。

Object的JSON序列化结果是标准的字面量形式，而映射对象的JSON处理结果是关联数组的形式表达。这仲序列化结果的意义在于通过网络等方式反馈了结果后，通过JSON.parse方法直接将解释结果传入Map构造函数中来得到 正确的映射对象。

WeakSet 和 WeakMap

Weak类型的数据似乎是可以被用来保证内存安全，他们的键值对只能是引用类型，简单的值类型将无法加入其中。
WeakSet有以下三个特性

  1. 不能包含值类型的元素，否则会抛出TypeError
  2. 不能包含无引用的对象，否则会自动清出集合

    也就是说当WeakSet中任一元素的引用被全部解除后，该元素就会被移出集合

  3. 无法被探知大小，也无法被探知其中包含的元素

WeakMap和WeakSet很类似，只不过是WeakMap的键会被检查引用

JavaScript中的This和其他编程语言中的This有那么一点区别，并不会像Java中的This一样指向本身，而是由函数运行时的环境来决定指向什么….

什么是This

​ This是函数作用域中的关键字，指向某个对象。不同的执行上下文会导致不同的this指向。

​ 上面的栗子中，看上去“this.count” 应该是指向fun自己的count变量，然而实际上函数执行的时候this指向了window

​ 当给函数明确调用对象的时候，this就指向的是调用的对象。

​ 由此看出This不同于函数，是在运行时绑定的而不是定义的时候进行绑定。this依赖于调用函数的上下文，和函数被调用的方式相关

This绑定规则

​ this包含四种绑定规则，分别为默认绑定、隐式绑定、明确绑定以及New 绑定，在Es6箭头函数中的this会比较特殊，指向它的上一层作用域。

默认绑定

​ 默认绑定顾名思义就是在没有其他绑定规则适用的情况下采用此种规则进行绑定。第一个栗子就是对this进行默认绑定，当函数不加任何修饰被独立调用的时候采用默认绑定，将this绑定到全局window上；因为在全局作用域中并没有“count”变量，所以结果为undefined。

​ 在非严格模式下默认绑定会将this指向window，但是在严格模式下这是不被允许的。

​ 另外，只有在以“严格模式”运行的函数中定义this并进行默认绑定才会出现语法错误

隐式绑定

​ 当this所在函数是被一个对象所拥有的时候，将会对this和该对象进行绑定。例如上面第二个例子中fun( )属于obj，也就是说被obj这个对象所拥有，那么就会将obj绑定到this上: this.count 等价于 obj.count 自然就得到了1

​ 需要注意的是只有对象属性引用链的最后会对this造成影响，就如同上面的代码打印的不是obj2而是obj的count。

​ 隐式绑定有时也会丢失，上面代码中的copy引用了obj中的fun方法，所以执行copy( )实际上就相当于不加修饰的直接调用了 fun( )，从而this指向了全局

明确绑定

​ 明确绑定即使用call、apply 方法指定你想要的this，这两个方法的第一个参数都用于绑定原函数中的this，后面的参数一般用于将实参传递给原函数

New绑定

​ 在使用new 关键字调用方法的时候，会产生出一个新的对象，这个新创建的对象将会被设置为调用函数的this绑定. new fun( )时 产生一个全新的对象“xxx” 并将”xxx” 与被调用的fun()中的this进行绑定

绑定规则的优先级

​ 绑定规则的优先级由高到低分别为：new绑定 >> 明确绑定 >> 隐式绑定 >> 默认绑定

  • 被new调用： this相当于新构建的对象
  • 被call、apply或者bind调用：this相当于指定的对象
